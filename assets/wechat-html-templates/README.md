# 公众号 HTML 模板包

这四个文件都是可直接双击预览的 HTML。先选择文章类型，再替换文件顶部的 `article` 或 `promo` 数据块；不要直接改页面结构。

| 文件 | 场景 | 图片要求 |
| --- | --- | --- |
| `01-magazine-longform.html` | 深度拆解、方法论、长文 | 1 张封面 + 2 张文内图 |
| `02-tutorial-checklist.html` | 清单、教程、步骤文 | 1 张封面 + 2 张步骤图 |
| `03-story-opinion.html` | 观点、复盘、故事文 | 1 张封面 + 2 张情绪/证据图 |
| `04-promo-card.html` | 文章或 Skill 的宣传配图 | 1 张 1200×630 横图 |

## 使用流程

1. 根据文章选一个模板，替换顶部数据块中的标题、摘要、段落和图片 URL。
2. 每篇文章至少准备 1 张封面、2 张与正文有直接关系的图片；为每张图写 `alt`、`caption`、`source` 和 `image_prompt`。
3. 用浏览器打开 `01`、`02` 或 `03`，点击“复制公众号兼容排版”。它会把当前渲染结果转换成没有 `script`、没有 `style` 标签、且全部样式内联的富文本，再粘贴到公众号正文。发布前仍须在公众号后台预览。
4. 同步填写 `04-promo-card.html`，导出 1200×630 图片，用于 X、朋友圈、社群或公众号文章预热。

`wechat-paste-test.html` 是不含脚本和样式表的独立测试文章：在浏览器打开后全选复制，粘贴到公众号新建图文中，先确认标题、引用框和列表的保留情况。图片请在公众号后台上传或替换；不要依赖外部图片链接。

## 图片原则

- 图片必须服务内容：实操文放流程截图或关键结果，观点文放事实证据或真实场景，不用无关的“氛围图”填空。
- 优先使用自己拍摄、自己生成、已获授权或明确可商用的素材；保留来源说明。
- 发布前检查人物肖像、品牌素材和截图内的隐私信息。
